计算机网络

网络通信的三要素是什么

网络通信的三要素分别是协议、IP 地址和端口号。

IP 地址（Internet Protocol Address）就是互联网协议地址，它表示设备在互联网中的地址，是唯一的标识。IP 地址有两种表示方式，分别是 IPv4 和 IPv6。IPv4 由 4 个字节组成，采用点分十进制表示法，IPv6 由 16 个字节组成，采用冒分十六进制表示法。

端口号（port）是程序在设备中的唯一标识，分为周知端口、注册端口和动态端口。周知端口的范围为 0-1023，这些端口被预先定义的应用占用，注册端口的范围为 1024-49151，这些端口被分配给用户进程或某些应用程序，动态端口的范围为 49152-65535，这些端口通常不会固定分配给某个进程，而是动态进行分配。一个设备中不能出现两个端口号一致的程序。

网络通信协议是设备在网络之间进行通信时事先规定的连接和数据传输规则。

UDP 和 TCP

UDP 和 TCP 是传输层的两个通信协议。

UDP即User Datagram Protocol（用户数据报协议），UDP的特点为无需连接就可以发送数据，速度快但易丢失数据，数据以包的形式传输，一个包最多发送64KB的数据。（无连接、不可靠）

TCP即Transmission Control Protocol（传输控制协议），TCP的特点为发送数据前需要进行连接，速度慢但不易丢失数据，数据以流的形式传输，没有大小限制。(面向连接、可靠通信)

UDP常用于直播、视频通话等对于速度要求高、数据完整性要求较低的场景，TCP常用于下载、扫码支付等对于速度要求较低、数据完整性要求较高的场景。

浏览器工作原理

ref：参考来源open in new window

影响 web 性能的两大主要原因：等待资源加载时间、浏览器单线程执行。

web 性能优化的目标：缩短导航（输入 url、点击链接、提交表单）所花费的时间。

  1. 浏览器根据用户输入的 url 在缓存中检索域名对应的 IP 地址，如果是未访问过的域名，浏览器会向名称服务器发送 DNS 查询请求。
  2. 获取到服务器的 IP 地址后，浏览器通过 TCP 三次握手与服务器建立连接。此外，为了在 HTTPS 上建立安全连接，还需要进行 TLS 协商（五次），决定什么密码将会被用来加密通信。
  3. 当浏览器与服务器建立连接后，浏览器代表用户向服务器发送一个 HTTP 的 get 请求（通常是一个 HTML 文件），服务器收到请求后，它将使用相关的响应头和 HTML 内容进行回复。
  4. 一旦浏览器接收到服务器响应的数据的第一块后，它就可以开始解析收到的信息，解析是指浏览器将通过网络接收的数据转换为 DOM（文档对象模型）和 CSSOM（CSS 对象模型）。
  5. 渲染步骤包括样式、布局、绘制。在解析步骤中创建的 CSSOM 树和 DOM 树组合成一个 Render 树，然后用于计算每个可见元素的布局，然后将其绘制到屏幕上。
