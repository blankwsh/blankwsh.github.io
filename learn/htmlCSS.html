渐进增强和优雅降级

两者都是前端开发的策略。

渐进增强就是一开始就针对低版本浏览器进行页面构建，完成基本的功能，然后再针对高级浏览器追加功能以到达更好的用户体验。

优雅降级就是一开始就构建站点的完整功能，再根据不同浏览器进行测试和修复，使其可以在低版本浏览器正常浏览。

联系与区别：两者都需要建立一个基准线，渐进增强是在基准线之上做功能增强；优雅降级则是在基准线之下做兼容。

.transition { /*渐进增强写法*/
  -webkit-transition: all .5s;
     -moz-transition: all .5s;
       -o-transition: all .5s;
          transition: all .5s;
}
.transition { /*优雅降级写法*/
          transition: all .5s;
       -o-transition: all .5s;
     -moz-transition: all .5s;
  -webkit-transition: all .5s;
}

position 属性的值有哪些及其区别

position的属性值有static、relative、absolute、fixed、sticky，默认情况下不设置position的属性值均为static。

当元素的position设置为relative时，元素先放置在未添加定位时的位置，再在不改变页面布局的前提下调整元素位置（因此会在此元素未添加定位时所在位置留下空白）。

当元素的position设置为absolute时，元素会被移出正常文档流，并不为元素预留空间，通过指定元素相对于最近的非 static 定位祖先元素的偏移，来确定元素位置。绝对定位的元素可以设置外边距（margins），且不会与其他边距合并。

当元素的position设置为fixed时，元素会被移出正常文档流，并不为元素预留空间，而是通过指定元素相对于屏幕视口（viewport）的位置来指定元素位置。元素的位置在屏幕滚动时不会改变。打印时，元素会出现在的每页的固定位置。fixed属性会创建新的层叠上下文。

当元素的position设置为sticky时，元素根据正常文档流进行定位，然后相对它的最近滚动祖先（nearest scrolling ancestor）和 containing block（最近块级祖先 nearest block-level ancestor），基于 top、right、bottom 和 left 的值进行偏移。偏移值不会影响任何其他元素的位置。 该值总是创建一个新的层叠上下文（stacking context）。

总结：static、relative和sticky的元素不会移出正常文档流，absolute和fixed会被移出正常文档流。relative的偏移相对于原位置，absolute的偏移相对于最近的非static定位祖先元素，fixed的偏移相对于屏幕视口，sticky的偏移相对于最近的滚动祖先或块级祖先。

CSS盒子模型

box-sizing属性用于设置一个元素总宽度和总高度的定义方式，属性值可以是content-box（标准盒子模型）和border-box（IE盒子模型）。

content-box表示元素的总宽度width为内容物的宽度，总高度height为内容物的高度。

border-box表示元素的总宽度width为内容物的宽度+padding+border，总高度为内容物的高度+padding+border。

回流和重绘

回流（reflow）是指浏览器根据渲染树中元素的布局信息计算并确定每个元素在页面中的准确位置和大小。当页面中的布局发生变化时（例如添加、删除或修改元素的尺寸、位置、内容等），浏览器需要重新计算整个页面的布局，这个过程就是回流。回流会导致页面重新排列元素，对性能产生较大的影响。

获取offsetTop、offsetLeft、 offsetWidth、offsetHeight、scrollTop、scrollLeft、scrollWidth、scrollHeight、clientTop、clientLeft、clientWidth、clientHeight等值以及使用getComputedStyle方法都会触发回流，因为浏览器为了获取这些值需要进行及时运算。

重绘（repaint）是指当元素的可见样式发生改变（例如颜色、背景、阴影等）时，浏览器重新绘制受影响的元素。重绘并不涉及布局的计算，只是重新绘制元素的外观。相比于回流，重绘的开销较小。

减少回流重绘的方法：通过改变元素类名实现样式切换；多使用CSS动画代替JS动画；避免使用表格布局（多次回流）；使用变量缓存上述提及会触发回流的属性，再在变量上作计算；采取离线操作（先将欲操作的元素设置为display: none; 这样不会触发回流重绘）

隐藏页面元素

  1. display: none; 元素被隐藏，页面布局随之改变，且绑定在元素上的事件不能触发。（重绘且回流）
  2. visibility: hidden; 元素被隐藏，但页面布局不会改变，且绑定在元素上的事件不能触发。（只重绘不回流）
  3. opacity: 0; 元素被隐藏，但页面布局不会改变，且绑定在元素上的事件能够正常触发。（一般情况下会重绘但不回流）
  4. .hiddenBox {
    margin:0;     
    border:0;
    padding:0;
    height:0;
    width:0;
    overflow:hidden;
}
  5. position:absolute; 修改 left 和 top 将元素移出可视区域。

Flex布局

只需要在一个容器上使用 display: flex; 即可将该容器的布局方式修改为弹性布局。弹性布局提供了一种更便捷的布局方式，其布局与两条轴线密切相关，分别是主轴和交叉轴。

主轴由 flex-direction 定义，取值可以是 row row-reverse column column-reverse ，前两者会使得容器中的元素按照inline方向排列，后两者则会使容器中的元素按照block方向排列。交叉轴则是与主轴垂直的轴线。

Flex布局不会对文档的书写模式提供假设，不再假设一行文字是从文档的左上角开始向右书写的，新的行也不必出现在另一行的下方。row 和 row-reverse 的区别就在于起始线和终止线的不同。

Flex容器中的flex元素具有以下特点：元素排列为一行，元素从主轴的起始线开始排列；元素在主轴方向上不会被拉伸，但可以被缩小；如果又元素超出容器，不会换行而是会溢出；如果一些元素比其他元素高，那么元素会沿交叉轴被拉伸来填满它的大小。

对于Flex容器，当子元素的总宽度大于容器的宽度，子元素不会自动换行，会自动对子元素进行缩小以适应容器，要想实现子元素换行，则需要将 flex-wrap 属性的值从默认的 nowrap 修改为 wrap。

flex-flow 是 flex-direction 和 flex-wrap 的简写属性。

flex 是 flex-grow、flex-shrink（弹性收缩因子） 和 flex-basis （弹性增长因子）的简写属性，这三个属性都是用于操作Flex容器中的可用空间。其中， flex-basis 定义了元素的空间大小，如果取值为 auto，则会根据元素的width设置元素的大小，如果没有设置 width ，则根据元素的内容设置元素的大小，如果取值为 content ，则忽略width，只根据元素的内容设置元素的大小； flex-grow 的值如果是一个正整数，则flex元素会以 flex-basis 为基础，沿主轴方向增长尺寸，这会使该元素延展，并占据此方向轴上的可用空间，flex-grow 属性可以按比例分配正可用空间。如果第一个元素 flex-grow 值为 2，其他元素值为 1，则第一个元素将占有 2/4，另外两个元素各占有 1/4。如果我们的容器中没有足够排列 flex 元素的空间（负可用空间），那么可以把 flex 元素flex-shrink 属性设置为正整数来缩小它所占空间到flex-basis以下。（负可用空间消除期间弹性盒子会阻止小的元素缩小到 0。这些元素会以 min-content （即最长单词的长度）的大小进行铺设）

align-items 可以设置元素在交叉轴的对齐方式，默认值是 stretch，表示所有flex元素会默认被拉伸到最高元素的高度，可选值有 flex-start、flex-end 和 center。

justify-content 可以设置元素在主轴的对齐方式，默认值是 flex-start，可选值有 stretch、flex-end、space-between、space-around 和 center。space-between 是先排列好元素，再将可用空间均分到元素之间（元素紧贴起始线和终止线），space-around 是等分可用空间并使得元素的左右间距相等（元素与起始线的距离是元素与元素之间距离的一半）。

order 属性用于修改元素的顺序，gap 属性用于设置元素间的间隙。

相对单位和绝对单位

  1. rem：相对于根元素（html）的字体大小而计算的单位。
  2. em：相对于元素的字体大小而计算的单位。当一个元素没有显式设置字体大小时，1em 等于其父元素的字体大小。当一个元素显式设置字体大小时，1em 等于该元素自身的字体大小。
  3. vh：相对于视口高度的单位，视口高度指的是可视区域的高度。
  4. vw：相对于视口宽度的单位，视口宽度指的是可视区域的宽度。
  5. %：相对于父元素的百分比单位。（特别地，translate和background-size是相对于自身，以及border-radius）
  6. px：屏幕上的一个像素
  7. em：1em=16px，em 会继承父级元素的字体大小
html {font-size: 62.5%;  } /*  公式16px*62.5%=10px  */

清除浮动

浮动元素会脱离当前文档流，以至于计算父元素高度时不会把其算进去，容易造成父元素高度塌陷。为了让父元素的高度计算正常，我们经常要手动清除浮动。

  1. 空div配合 clear: both;
  2. 伪元素选择器:after配合 clear: both;

  content: "";
  clear: both;
  display: block;
}
  3. 父元素overflow: auto; 创建BFC

动画

动画是指元素从一种样式转变为另一种样式的过程。常见的实现动画的方式有transition（实现渐变动画）、transform（实现转变动画）和animation（实现自定义动画）。

transition（过渡）

transition是transition-property、transition-duration、transition-timing-function和transition-delay这四个属性的缩写。其中，transition-property用于指定应用过渡的属性，如果取值为all，则表示该过渡将应用于所有的属性上；transition-duration指定过渡动画的持续时间，需要注意的是，使用时需要加上单位s/ms，其默认值为0s；transition-timing-function用于指定完成过渡效果的速度曲线，默认值为ease（缓慢开始、急剧加速、缓慢结束），还可以取linear（匀速）、ease-in（缓慢开始）、ease-out（缓慢结束）、ease-in-out（缓慢开始、缓慢结束），cubic-bezier(n,n,n,n)（贝塞尔曲线），steps(n,direction)（步长函数），更多相关参见：https://juejin.cn/post/7131673844552564767 ；transition-delay定义过渡效果开始作用前的等待时间，其他内容与transition-duration一致。多个transition之间用逗号分隔。可以通过监听transitioned事件判断过渡是否完成。

需要注意的是，过渡效果对display、visibility、position、border、outline的变化无效（测试visibility是动画效果无效，但时间有效，似乎被当作是transition-delay），这些属性的变化都是立刻完成的，还有对于涉及margin: auto; 的变化也无效，因为过渡只能应用于具体的长度或百分比值。

transform（变换）

transform可以在不影响正常文档流的情况下改变作用内容的位置，实现对元素的平移、旋转、倾斜和缩放。变换函数可以是translate、rotate、skew、scale，其中，除了skew不能单独作为属性使用外，其他三个属性均可。多个transform函数间用空格分隔。

translate的平移方式：正值向右下偏移，负值向左上偏移，取值也可以是百分值，百分比相对于自身。

rotate的旋转方式：取值为角度deg，正值表示顺时针旋转，负值表示逆时针旋转。

skew的倾斜方式：对于skewx，如果是正值x，则表示左边界与x方向倾斜呈x角度，如果是负值，则表示右边界与x方向倾斜呈x角度。skewy同理。

scale的缩放方式：与其他变换不同的是，如果scale只有一个参数，那么缩放会同时在两个方向上生效。如果想要指定单一方向的缩放，需使用scaleX、scaleY。

transform-orgin是用于调整变换中心的属性，属性值可以指定两个关键词作为定位点，如top、bottom、left、right和center，也可以是一般的长度单位值如px、%等。

animation（动画）

animation是animation-name、animation-duration、animation-timing-function、animation-delay、animation-iteration-count、animation-direction、animation-fill-mode和animation-play-state的缩写。

animation-name用于指定动画关键帧的名称，对应于keyframes的名称；

animation-duration则是定义动画的持续时间；

animation-tming-function则是用于定义动画效果的速度曲线；

animation-delay用于指定完成动画效果开始作用前的等待时间；

animation-iteration-count用于指定动画在结束前应播放的次数，这里的次数可以是小数，还可以是关键字infinite；

animation-direction设置动画的播放方向，当取值是normal时，表示动画正向播放（从起始状态开始播放），当取值是reverse时，表示动画反向播放（从终止状态开始播放），这里需要注意动画的速度曲线函数也会反向，如ease-in会变为ease-out，当取值是alternate时，动画将会正反交替播放，第一次是正向播放，当取值是alternate-reverse时，动画也会正反交替播放，但第一次是反向播放。

animation-fill-mode设置动画在执行之前和执行之后应该如何应用样式，当取值是none时，表示初始样式与动画的初始样式无关，仅与自身初始样式有关（区别于取值为backwards），且终止样式也为自身初始样式（区别于取值为forwards）；当取值是backwards，表示初始样式为动画的初始样式，终止样式为自身初始样式；当取值为forwards，表示终止样式为动画的终止样式，初始样式为自身初始样式；当取值为both时，表示初始样式和终止样式分别对应动画的初始样式和终止样式，即自身初始样式失效。

animation-play-state设置动画是运行还是暂停的。当取值为running时，表示动画正在运行；当取值为paused，表示动画已被停止。注意，恢复暂停的动画将从暂停时停止的位置开始播放，而不是从动画序列的开头重新开始播放。

动画事件：animationstart（动画开始）、animationed（动画结束）、animationiteration（动画循环）

响应式设计

响应式设计是一种网页开发与设计的方法，它的核心理念是，一个网站应当能够根据用户的设备类型和屏幕尺寸自动地调整布局、内容的展示方式，以适应不同的浏览环境。

响应式设计的基本实现原理是通过媒体查询检测用户的设备屏幕尺寸进行相应的处理。为了处理移动端，页面头部必须有meta声明viewport。

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no”>

其中，name="viewport"指定了元标签的名称，即该元标签用于设置视口属性；content是视口属性的具体内容，width=device-width将视口的宽度设置为设备的宽度，确保网页在移动设备上不会被放大或缩小以适应屏幕；initial-scale=1设置初始的缩放比例为1，即不进行任何缩放；maximum-scale=1设置允许缩放的最大比例为1，防止用户通过手动缩放改变网页的显示；user-scalable=no禁止用户手动缩放。

除了媒体查询外，响应式设计还可以通过相对单位或弹性布局等实现。

滚动视差

滚动视差是指多层背景以不同的速度滚动，形成立体的运动效果。通常情况下，滚动视差是通过JavaScript实现的，当然也可以用CSS来实现。

我们可以将网页划分成三层，最底层的是背景层，中间的是内容层，最上面的是视口层。通常情况下，视口层在移动时，内容层和背景层也会随之移动。我们可以在视口层移动时，固定背景层，只让内容层随之移动，从而实现滚动视差的效果。

要想固定住背景层，我们可以使用background-attacment属性，该属性决定背景图像的位置是在视口内固定，或者随着包含它的区块滚动，主要有三个取值，分别是local、scroll和fixed。fixed表示背景相对于视口固定；scroll表示背景相对于元素本身固定，而不是随着它的内容滚动；local表示背景相对于元素的内容固定。如果一个元素拥有滚动机制，背景将会随着元素的内容滚动，并且背景的绘制区域和定位区域是相对于可滚动的区域而不是包含他们的边框。

如果我们将网页划分成五等分，其中1、3、5正常显示，对2和4的背景设置background-attachment: fixed，即可将背景固定在视口上，从而实现滚动视差的效果。

另外还可以通过3D效果实现，具体原理如下：

我们给容器设置上transform-style: preserve-3d和perspective: xpx，那么处于这个容器的子元素就将位于3D空间中。

再给子元素设置不同的transform: translateZ()，这个时候，不同元素在 3D Z轴方向距离屏幕（我们的眼睛）的距离也就不一样。
