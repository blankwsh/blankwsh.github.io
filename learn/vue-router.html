Vue Router

Vue Router 是 Vue 的官方路由，通过使用 Vue+Vue Router 可以轻松地构建单页面应用。

router-link 和 router-view

<router-link>是 Vue Router 的一个自定义组件。<router-link>可以将页面导航到to属性指定的位置，它会被编译成一个带有href属性的<a>标签，与<a>标签不同的是，<router-link>可以使得 Vue Router 在不重新加载页面的情况下更改 URL，处理 URL 的生成和编码。

<router-view>是 Vue Router 的一个自定义组件，是路由的出口，它将显示<router-link>中 URL 对应的路由组件。

使用 Vue Router 创建一个单页面应用的基本过程

  1. 通过npm install vue-router@4在项目中安装 Vue Router
  2. 创建路由组件（路由组件通常会单独存放在一个 views 文件夹中）
  3. 创建路由实例（路由信息通常会单独存放在一个 router 文件夹中）
  • 从vue-router中引入createRouter用于创建路由实例
import { createRouter } from "vue-router";
  • 引入路由组件
import Home from "../views/Home.vue";
import About from "../views/About.vue";
  • 定义路由规则
const routes = [
  {
    path: "/",
    component: Home,
  },
  {
    path: "/about",
    component: About,
  },
];
  • 向createRouter方法传入路由规则routes创建路由实例，并向外暴露这个路由实例
export default createRouter({
  history: createWebHashHistory(),
  routes,
});
  1. 在 Vue 实例中引入路由实例，通过 Vue 实例上的use方法使 Vue 实例支持路由
import { createApp } from "vue";
import App from "./App.vue";
import router from "./router";
createApp(App).use(router).mount("#app");
  1. 使用路由
  • 引入自定义组件<router-link>和<router-view>
  • 使用<router-link>创建路由导航，使用<router-view>指示路由出口
<script setup>
import { RouterLink, RouterView } from "vue-router";
</script>
<template>
  <nav>
    <RouterLink to="/">Home</RouterLink>
    <RouterLink to="/about">About</RouterLink>
  </nav>
  <RouterView />
</template>

路由（router）和当前路由（route）

在选项式API中，可以通过this.$router和this.$route访问路由和当前路由，而在组合式API的setup函数中，需要通过useRoute和useRouter进行访问（模板中仍然可以通过$进行访问）。

路径参数（params）和动态路由

在Vue Router中，可以在路径中使用路径参数来实现动态路由，路径参数用:表示，如下所示：

const routes = [
    {
        path: '/:id',
        component: Home
    },
    {
        path: '/:name',
        component: About
    }
]

路径参数可以通过当前路由进行访问，即route.params。

当路由跳转时只有路径参数发生变化时，相同的组件实例会被复用，这也就意味着组件的生命周期钩子不会被重新调用。

在上面的路由规则中，路由组件的路径都是只有一个路由参数，这就会导致两个路由匹配完全相同的URL，除了可以添加静态部分进行区分外，还可以通过正则表达式对参数进行区分，如下所示：

const routes = [
    {
        // 括号中书写正则表达式，表示id的参数类型只能为数字
        // 注意字符串中的反斜杠会被视为转义符号，需要使用反斜杠将其转化为文本的反斜杠
        path: '/:id(\\d+)',
        component: Home
    },
    {
        // 非数字类型的参数就会被导航到About组件
        path: '/:name',
        component: About
    }
]

和正则表达式类似，路径参数也可以通过+，*，？来对参数的量词进行设置，如下所示：

const routes = [
    {
        // 表示至少需要一个id参数，可以匹配'122/123'、'1254'等路径
        path: '/:id(\\d+)+',
        component: Home
    }
]

sensitive和strict

默认情况下，路由不区分大小写，不区分尾部带有或是不带斜杠，'/users'、'/Users'、'/users/'、'/Users/'都会被导航到同一路由组件下，可以通过strict（区分斜杠）和sensitive（区分大小写）配置项进行修改，如下所示：

const routes = [
    {
        path: '/users',
        sensitive: true,    // '/Users'不可以被匹配
        strict: true,        // '/users/'不可以被匹配
        component: Home
    }
]

嵌套路由和命名路由

URL的片段往往对应了特定的嵌套组件结构，我们可以通过嵌套路由来表示这种关系。可以通过在路由中配置children来指定路由的子路由，子路由的配置项与普通路由保持一致，需要注意的是，以/开头的嵌套路径将会被视为根路径，因此子路由的路径选项不应以/开头。
通过为路由添加一个name属性，可以创建一个命名路由，需要注意的是，所有路由的命名必须是唯一的，如果存在多个同名路由，路由器只会保留最后一个，如下所示是一个命名路由的使用方式：

const routes = [
    {
        path: '/:user',
        name: 'home',
        component: Home
    }
]

<!-- 注意这里使用了对象式的to，需要使用v-bind进行绑定 -->
<router-link :to="{ name: 'home', params: { user: 'goh' } }"></router-link>
<!-- 相当于<router-link to="/goh"></router-link> -->

编程式导航

除了使用<router-link>创建a标签来定义导航链接外，还可以通过router的实例方法实现路由的导航。

router.push

router.push方法会向history记录栈添加一个新记录

router.replace

router.replace方法会取代history记录栈的当前记录。

router.go

该方法接收一个整数作为参数，表示在history栈中前进（正数）或后退（负数）几步。特别的，router.go(1)等价于router.forward()，router.go(-1)等价于router.back()
